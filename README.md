## Project Description
This project is a terminal-based application developed in Python to automate the registration and processing of customer orders for a distribution company. The system replaces manual spreadsheets and paper records with a structured, modular, and efficient digital solution.

The primary goal is to provide a tool for commercial operations managers to track customers, products, and daily sales while generating consolidated financial reports at the end of the workday.

##  Key Features
The system is divided into six core modules, each handled by a dedicated team member:
1. **Customer Registration:** Securely store customer IDs, names, and emails.
2. **Product Management:** Maintain a catalog of available products and prices.
3. **Order Processing:** Create transactions by linking customers and products with automatic total calculation.
4. **Order Consultation:** Real-time visualization of all recorded transactions.
5. **Revenue Analysis:** Real-time calculation of total income generated during the session.
6. **Consolidated Final Report:** A comprehensive summary including total orders, revenue per customer, and best-selling products.

##  Technical Constraints & Architecture
To ensure high-quality coding standards and data integrity, the project follows these strict technical rules:

* **Data Structures:** * **Dictionaries (`{}`):** Used for primary data storage (Customers and Orders) to allow fast lookups via unique IDs.
    * **Tuples (`()`):** Used for immutable product information (Name, Price) to ensure data consistency.
    * **Lists (`[]`):** Strictly **PROHIBITED** to demonstrate advanced knowledge of key-value pair management.
* **Modular Programming:** The code is organized into reusable functions. Each function is designed to receive parameters and return data, avoiding "side-effect" only functions.
* **Language:** Both the source code (variables, functions, comments) and documentation are written entirely in **English**.
* **Project Management:** Developed using **Agile practices** and tracked via **Azure DevOps**.
